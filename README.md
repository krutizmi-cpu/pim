# PIM — модуль базового каталога товаров

Этот этап проекта — первый модуль PIM: **базовый каталог товаров** для торговой компании.

## Проектный ориентир

- Основной метод и рамки разработки зафиксированы в файле [PIM_METHOD.md](PIM_METHOD.md).
- Оперативный контекст проекта, этапы и handoff для агентов: [PROJECT_CONTEXT.md](PROJECT_CONTEXT.md).
- Правила работы агентов и обязательные проверки: [AGENTS.md](AGENTS.md).
- Журнал заметных изменений и регрессий: [CHANGELOG.md](CHANGELOG.md).
- Перед любыми новыми доработками сверяйся с этим документом, чтобы не уходить в сторону.

## Что реализовано

- хранение каталога в SQLite (`data/catalog.db`);
- обязательные поля: `article` (уникальный), `name`;
- необязательные поля: `barcode`, `weight`, `length`, `width`, `height`, `supplier_url`, `image_url`, `description`;
- импорт каталога из Excel (1С) с обновлением по `article`;
- таблица `duplicate_candidates` для похожих `name` (>85%);
- Streamlit-раздел «Каталог товаров» с поиском и фильтрами;
- кнопка «Обогатить данные» как заглушка с логом в `data/enrichment.log`.
- AI-слой с настраиваемыми провайдерами (`OpenAI` / `OpenRouter` / `NVIDIA`) через OpenAI-compatible API.
- AI-функции в карточке:
  - генерация продающего названия товара;
  - генерация SEO-описания по названию/артикулу/атрибутам;
  - AI-подсказки для пустых Ozon-атрибутов с сохранением источника `ai`;
  - генерация 2 маркетинг-изображений (контекстный и цветной варианты) по промптам.
  - основной one-click сценарий в карточке теперь называется `Заполнить карточку автоматически` и запускает последовательность `Ozon -> supplier/web -> AI`.
- В каталоге:
  - массовый AI-пакет по фильтру: supplier/web enrichment + AI title + AI description + AI Ozon attributes.
  - основной UX теперь строится вокруг кнопок `Заполнить текущую страницу` / `Заполнить всю выборку фильтра`, а отдельные технические действия вынесены в дополнительные блоки.
- Постоянное хранение данных:
  - по умолчанию PIM старается использовать постоянную SQLite БД `~/.pim/catalog.db`;
  - если рядом найдены несколько баз, PIM не только выбирает, но и пытается объединить данные в `~/.pim/catalog.db`, чтобы не терялись товары, Ozon cache, mappings и template profiles;
  - импортные Excel и клиентские шаблоны сохраняются в `~/.pim/uploads/...`, а история импорта и AI-профили — в самой SQLite БД.
- Клиентские шаблоны:
  - клиент выбирается из базы, а не только вводится вручную;
  - нового клиента можно создать прямо в разделе `Клиентский шаблон`;
  - сохранённые Excel-шаблоны клиента можно открывать повторно из памяти по связке `клиент + категория`.
  - для `Sportmaster` появился category-specific import: Excel-шаблоны из личного кабинета можно один раз загрузить в память PIM, после чего категории, обязательность полей, mapping rules и template profiles живут в БД как отдельные профили `sportmaster:<attr_class_id>`.
- Новый shell интерфейса:
  - боковая навигация по главным разделам вместо длинной грядки вкладок;
  - единая workspace-шапка с состоянием памяти PIM;
  - более чистый light UI с акцентом на рабочие KPI, каталог и шаблоны;
  - общие настройки PIM вынесены в отдельный раздел `Настройки`, чтобы не засорять `Каналы` и `Каталог`.
- UX каталога и карточки:
  - `Каталог` показывает более операционный список товаров: этап карточки, статус парсинга, фото и заполненность ядра;
  - из каталога можно сразу открыть выбранный товар в `Карточка` без дополнительного поиска;
  - `Карточка` получила верхнюю сводку состояния товара и более компактную форму, сгруппированную по блокам `Идентификация`, `Категория и Ozon`, `Логистика`, `Фото и описание`.
  - массовые действия теперь можно запускать не только по странице или фильтру, но и по вручную выбранному shortlist товаров;
  - в `Каталог` и `Карточка` видна базовая готовность товара под Ozon и лучший клиентский шаблон;
  - фото в карточке показываются крупнее, с отдельным списком ссылок, а в каталоге есть быстрый превью-блок по выбранному товару.
- Клиентские шаблоны и bulk-export:
  - в `Клиентский шаблон` теперь есть единый блок `Пачка товаров для шаблона`;
  - можно подтянуть shortlist из `Каталог` и использовать его для preview, readiness, gap-анализа и финальной выгрузки;
  - readiness пачки товаров показывается до скачивания файла, чтобы выгрузка была более предсказуемой.
- Backup-память:
  - после критичных операций (`Импорт`, сохранение клиентского профиля/мэппинга, ключевые Ozon sync/restore) PIM фиксирует SQLite backup в `~/.pim/backups/db/`;
  - snapshot Ozon сохраняется не только на скачивание, но и во внутреннюю память сервиса в `~/.pim/backups/ozon/`;
  - сохранённые Ozon snapshot можно повторно открыть и восстановить без повторной загрузки файла с компьютера.
- Detmir API как клиентский overlay:
  - в `Каналы` появился read-only блок `Детский Мир API`;
  - можно сохранить `client_id/api_key`, проверить соединение и синхронизировать Detmir schema и карточки;
  - в память PIM сохраняются кэши `detmir_category_cache`, `detmir_attribute_cache`, `detmir_attribute_value_cache`, `detmir_product_cache`;
  - Detmir не становится вторым master-ядром: его категории и атрибуты подтягиваются как клиентские требования поверх Ozon-эталона;
  - для выбранной Detmir-категории можно импортировать channel requirements и стартовые mapping rules в PIM;
  - в `Карточка` есть отдельный Detmir-flow: подобрать/исправить категорию, заполнить gaps, посмотреть readiness и payload.

## Инструкция по разделам

1. `Импорт`
- Загрузить Excel поставщика.
- Выбрать режим: автоопределение или ручной (`лист + строка заголовка`).
- При необходимости задать `Поставщик по умолчанию` и `Шаблон URL` (`{article}`, `{supplier_article}`, `{code}`, `{name}`).
- Выполнить импорт и проверить последнюю партию.

2. `Каталог`
- Использовать постраничный режим (`Страница`, `◀ Назад`, `Вперед ▶`, `Размер страницы`).
- Фильтровать по supplier parsing (`Есть supplier_url`, `Не парсено`, `Ошибка`, `Успех`).
- `Категория` и `Поставщик` выбирать из выпадающих списков БД.
- Глобальные настройки парсинга больше не редактируются здесь: в каталоге оставлены только параметры текущего запуска.
- Запускать массовые операции по текущей странице:
  - supplier enrichment;
  - автопривязка к Ozon категориям.
- При необходимости подготовить архив фото:
  - `Подготовить ZIP фото текущей страницы`
  - `Подготовить ZIP фото всей выборки`
- Использовать блок `Массовое изменение данных`:
  - назначать поставщика и URL template;
  - менять `category / base_category / subcategory / brand`;
  - применять к текущей странице или ко всей выборке по фильтру;
  - режим `только пустые поля` для безопасного дозаполнения.

3. `Карточка`
- Ручное дозаполнение мастер-товара.
- Парсинг поставщика (мягкий/force).
- Просмотр и правка Ozon category (`description_category_id`, `type_id`, `path`, `confidence`).
- Просмотр и правка Detmir category (`category_id`, `path`, `confidence`) с отдельным readiness-блоком.
- Контроль источников данных (manual/supplier/client и др.).
- Поиск сертификатов/деклараций ФСА:
  - поиск по юрлицу поставщика, виду товара и `ТН ВЭД`;
  - разбор найденной карточки/документа ФСА;
  - сохранение реквизитов документа и PDF в карточку товара.
- AI-контент:
  - сгенерировать SEO-описание;
  - найти значения для пустых Ozon-атрибутов;
  - подготовить/запустить генерацию 2 изображений.

4. `Атрибуты`
- Управление master/channel атрибутами.
- Хранение значений атрибутов товара (`product_attribute_values`).

5. `Клиентский шаблон`
- Загрузка Excel-шаблона клиента.
- Автоматический матчинг колонок + ручная корректировка.
- Конвертации единиц (`cm/mm/m`, `kg/g/lb`, `inch/cm`) и media transforms.
- Сохранение `mapping rules` и `template profiles` для повторного экспорта.
- Кнопка добавления новых несматченных характеристик в master-атрибуты.
- Для `Sportmaster`:
  - можно импортировать пакет категорийных Excel-шаблонов прямо в память PIM;
  - категория профиля берётся из `Атрибутный класс id` шаблона (`sportmaster:<attr_class_id>`);
  - PIM сохраняет в память не только сам `.xlsx`, но и обязательность колонок, инструкции, справочные значения и стартовые mapping rules;
  - повторяющиеся колонки вроде `Вид спорта*`, `Вид спорта*.1`, `Вид спорта*.2` сохраняются раздельно и не схлопываются.
- Для Wildberries category-шаблонов поддерживается авто-паттерн:
  - лист `Товары`,
  - шапка в строке 3,
  - пропуск строк-инструкций,
  - устойчивое чтение даже при битых `dataValidation` в xlsx.

6. `Ozon`
- Синхронизация дерева категорий/атрибутов/справочников Ozon.
- Ozon используется как эталон структуры карточки.
- Контроль покрытия товара под выбранную Ozon-категорию.
- Snapshot/restore:
  - подготовленный snapshot кэша Ozon одновременно доступен на скачивание и сохраняется во внутреннюю память сервиса;
  - из UI можно выбрать один из последних серверных snapshot и восстановить его без нового upload.

7. `Каналы`
- Поддержка channel requirements и mapping rules.
- Повторное использование правил при новых выгрузках.
- Detmir API:
  - сохранение `client_id/api_key` в системных настройках или чтение key из env;
  - read-only sync дерева категорий, категорий с атрибутами, справочных значений и карточек товаров;
  - импорт Detmir category requirements в overlay-слой PIM (`channel_attribute_requirements` + стартовые `mapping rules`);
  - после sync основной рабочий сценарий менеджера продолжается в `Карточка`, а не в техническом API-блоке.

8. `Настройки`
- `AI`:
  - активировать сохранённый профиль одним нажатием;
  - сохранить активную конфигурацию для всего PIM;
  - быстрая проверка провайдера и модели без тяжёлого chat-timeout на NVIDIA.
- `Парсинг`:
  - стратегия enrichment;
  - ограничения fallback;
  - пороги релевантности.
- `Фото`:
  - `Public base URL для фото`, если поставщик или 1С отдают локальные пути вместо web-ссылок.

## Сертификаты ФСА

- Для стабильной работы поиска лучше сохранить у поставщика `Юрлицо поставщика` во вкладке `Импорт -> Профили поставщиков`.
- В `Карточка -> Сертификаты / декларации ФСА` система использует:
  - юрлицо поставщика;
  - вид товара;
  - код `ТН ВЭД`;
  - при необходимости ручной URL документа.
- В карточке сохраняются:
  - номер документа;
  - дата начала;
  - дата окончания;
  - регистрирующий орган;
  - заявитель;
  - PDF-файл, если он доступен.
