# PIM — модуль базового каталога товаров

Этот этап проекта — первый модуль PIM: **базовый каталог товаров** для торговой компании.

## Проектный ориентир

- Основной метод и рамки разработки зафиксированы в файле [PIM_METHOD.md](PIM_METHOD.md).
- Перед любыми новыми доработками сверяйся с этим документом, чтобы не уходить в сторону.

## Что реализовано

- хранение каталога в SQLite (`data/catalog.db`);
- обязательные поля: `article` (уникальный), `name`;
- необязательные поля: `barcode`, `weight`, `length`, `width`, `height`, `supplier_url`, `image_url`, `description`;
- импорт каталога из Excel (1С) с обновлением по `article`;
- таблица `duplicate_candidates` для похожих `name` (>85%);
- Streamlit-раздел «Каталог товаров» с поиском и фильтрами;
- кнопка «Обогатить данные» как заглушка с логом в `data/enrichment.log`.

## Инструкция по разделам

1. `Импорт`
- Загрузить Excel поставщика.
- Выбрать режим: автоопределение или ручной (`лист + строка заголовка`).
- При необходимости задать `Поставщик по умолчанию` и `Шаблон URL` (`{article}`, `{supplier_article}`, `{code}`, `{name}`).
- Выполнить импорт и проверить последнюю партию.

2. `Каталог`
- Использовать постраничный режим (`Страница`, `◀ Назад`, `Вперед ▶`, `Размер страницы`).
- Фильтровать по supplier parsing (`Есть supplier_url`, `Не парсено`, `Ошибка`, `Успех`).
- `Категория` и `Поставщик` выбирать из выпадающих списков БД.
- Запускать массовые операции по текущей странице:
  - supplier enrichment;
  - автопривязка к Ozon категориям.
- Использовать блок `Массовое изменение данных`:
  - назначать поставщика и URL template;
  - менять `category / base_category / subcategory / brand`;
  - применять к текущей странице или ко всей выборке по фильтру;
  - режим `только пустые поля` для безопасного дозаполнения.

3. `Карточка`
- Ручное дозаполнение мастер-товара.
- Парсинг поставщика (мягкий/force).
- Просмотр и правка Ozon category (`description_category_id`, `type_id`, `path`, `confidence`).
- Контроль источников данных (manual/supplier/client и др.).

4. `Атрибуты`
- Управление master/channel атрибутами.
- Хранение значений атрибутов товара (`product_attribute_values`).

5. `Клиентский шаблон`
- Загрузка Excel-шаблона клиента.
- Автоматический матчинг колонок + ручная корректировка.
- Конвертации единиц (`cm/mm/m`, `kg/g/lb`, `inch/cm`) и media transforms.
- Сохранение `mapping rules` и `template profiles` для повторного экспорта.
- Кнопка добавления новых несматченных характеристик в master-атрибуты.
- Для Wildberries category-шаблонов поддерживается авто-паттерн:
  - лист `Товары`,
  - шапка в строке 3,
  - пропуск строк-инструкций,
  - устойчивое чтение даже при битых `dataValidation` в xlsx.

6. `Ozon`
- Синхронизация дерева категорий/атрибутов/справочников Ozon.
- Ozon используется как эталон структуры карточки.
- Контроль покрытия товара под выбранную Ozon-категорию.

7. `Каналы`
- Поддержка channel requirements и mapping rules.
- Повторное использование правил при новых выгрузках.
