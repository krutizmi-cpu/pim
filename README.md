# PIM — модуль базового каталога товаров

Этот этап проекта — первый модуль PIM: **база товаров**, которая сохраняется и используется для дальнейшей работы.

## Что сейчас умеет

- хранить каталог в SQLite (`data/catalog.db`);
- загружать базовый каталог из Excel/1С;
- создавать новые товары и обновлять существующие по `article`;
- искать возможные дубли по похожему `name` (>85%);
- показывать каталог в Streamlit с поиском и фильтрами;
- выгружать текущую выборку в Excel;
- отдавать **готовый шаблон загрузки** прямо из интерфейса;
- запускать заглушку «Обогатить данные» с логом в `data/enrichment.log`.

## Какой шаблон грузить

Минимально обязательные поля:

- `article` — уникальный артикул
- `name` — наименование товара

Необязательные поля:

- `barcode`
- `weight` — вес в кг
- `length` — длина в см
- `width` — ширина в см
- `height` — высота в см
- `supplier_url` — ссылка на товар у поставщика/производителя
- `image_url` — ссылка на фото
- `description` — базовое описание

Можно использовать и русские названия колонок:

- `Артикул`
- `Наименование`
- `Штрихкод`
- `Вес`
- `Длина`
- `Ширина`
- `Высота`
- `Ссылка поставщика`
- `Фото`
- `Описание`

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Импорт из командной строки

```bash
python import_1c_catalog.py catalog.xlsx
```

## Как работать

1. Открой приложение.
2. В блоке «Загрузить каталог из Excel» скачай шаблон.
3. Заполни минимум `article` и `name`.
4. Загрузи файл обратно в систему.
5. Каталог сохранится в базе и будет использоваться дальше для обогащения, наполнения карточек и выгрузок.
