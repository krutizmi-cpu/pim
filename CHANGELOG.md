# CHANGELOG.md

## 2026-04-27

### `pending` Fix partial Ozon requirements import, card filter reset, and public media export

- Исправлен разрыв между полным Ozon cache и фактическими атрибутами товара/категории в PIM.
- Причина:
  - если для пары `ozon:description_category_id:type_id` в `channel_attribute_requirements` уже были записи, система считала категорию полностью импортированной и не добирала недостающие Ozon-атрибуты после более полного sync/snapshot;
  - в `Карточка` кнопка `Сброс` меняла `st.session_state` после создания widget и вызывала `StreamlitAPIException`;
  - фото из локальных путей (`\\\\fs03\\...jpg`) не превращались в публичные URL для клиентских выгрузок и ZIP.
- Что изменено:
  - `ensure_ozon_requirements_for_product_category` теперь сравнивает объём `channel_attribute_requirements` с `ozon_attribute_cache` и добирает недостающие атрибуты вместо раннего выхода;
  - в `Атрибуты` добавлена диагностика по выбранной Ozon-паре: сколько атрибутов в кэше Ozon и сколько требований реально есть в PIM;
  - `Карточка -> Сброс` переведён на безопасный `on_click` callback;
  - добавлены настройки `Public base URL для фото` в `Каналы`;
  - локальные пути к фото теперь конвертируются в публичные URL в карточке и в template/export flow;
  - добавлена подготовка ZIP-архивов фото для текущей страницы и всей фильтрации каталога, а также ZIP по одному товару.
- Что проверить вручную:
  - после `Подтянуть Ozon-атрибуты` число требований по выбранной `cat/type` догоняет число атрибутов в Ozon cache;
  - в `Карточка` поиск по товару и кнопка `Сброс` больше не падают;
  - после задания `Public base URL для фото` клиентская выгрузка отдаёт web-ссылки вместо `\\\\fs03\\...`;
  - ZIP фото по товару / странице / фильтру собирается с именами `АРТИКУЛ_1`, `АРТИКУЛ_2` и т.д.

### `pending` Backfill Ozon/channel attribute blocks from parsed product state

- Исправлен разрыв между “плоским” блоком карточки товара и блоком `Атрибуты для заполнения (Ozon и клиентские шаблоны)`.
- Причина:
  - supplier parsing обновлял основные поля товара и часть master-атрибутов;
  - Ozon/category блок мог оставаться пустым, если Ozon-категория была назначена позже или если нужные значения уже были в карточке, но не переносились в category/channel-атрибуты.
- Что изменено:
  - добавлен сбор кандидатов из текущего состояния товара (`products` + `product_attribute_values`);
  - после supplier enrichment выполняется дополнительный backfill Ozon-атрибутов из уже найденных полей карточки;
  - при открытии карточки с Ozon-категорией система автоматически дозаполняет Ozon-блок из текущих данных товара;
  - в блоке `Атрибуты для заполнения (Ozon и клиентские шаблоны)` добавлена кнопка автозаполнения текущего блока из уже накопленных данных товара.
- Что проверить вручную:
  - если у товара уже заполнены `brand/weight/dimensions/...`, после открытия карточки часть Ozon-атрибутов должна подтянуться автоматически;
  - после `Спарсить поставщика` значения должны появляться не только в основном блоке товара, но и в Ozon-блоке;
  - для клиентского блока кнопка автозаполнения переносит уже найденные значения в текущий scope.

## 2026-04-23

### `pending` Normalize manual source tracking for product attributes

- Исправлена логика ручного сохранения атрибутов товара.
- Причина: в `Карточка` ручное редактирование атрибутов писало источник как `field_name=<code>`, тогда как остальная система использует формат `attr:<code>`. В `Атрибуты -> Быстрое редактирование` источник вообще не записывался.
- Что изменено:
  - ручное сохранение атрибутов в `Карточка` теперь пишет `field_name=attr:<code>`;
  - быстрое сохранение значения в разделе `Атрибуты` теперь тоже записывает manual-source в `product_data_sources`.
- Что проверить вручную:
  - изменить атрибут товара в `Карточка`;
  - изменить атрибут товара в `Атрибуты`;
  - в блоке источников данных видно `manual` для `attr:<код>`;
  - supplier/AI не должны перетирать вручную сохранённый атрибут без force.

### `pending` Materialize Ozon category attributes into product attribute slots

- Исправлен разрыв между Ozon category matching и фактическими атрибутами товара.
- Причина: после назначения Ozon-категории в PIM создавались `requirements` и записи в справочнике, но у самого товара часто не появлялись строки атрибутов в `product_attribute_values`.
- Что изменено:
  - добавлена материализация пустых Ozon-атрибутов товара по его `description_category_id + type_id`;
  - материализация запускается из `Каталог` при Ozon-привязке и при кнопке `Подтянуть Ozon-атрибуты`;
  - материализация запускается в `Карточка` при автопривязке Ozon и при открытии карточки товара с Ozon-категорией;
  - материализация запускается в `Атрибуты` для выбранного товара с Ozon-категорией.
- Что проверить вручную:
  - после назначения Ozon-категории у товара появляются пустые Ozon-атрибуты;
  - в `Карточка` блок `Атрибуты для заполнения` показывает Ozon-поля без ручного создания;
  - в `Атрибуты -> Атрибуты выбранного товара` видны материализованные Ozon-атрибуты выбранной категории.

### `pending` Handle anti-bot supplier domains without breaking fallback search

- Исправлен сценарий, когда supplier URL указывает на домен с антибот-защитой (например, `sportmaster.ru`) и массовое обогащение завершалось `успешно 0, ошибок N`.
- Причина: после ошибки `401 / anti-bot` система продолжала считать домен поставщика предпочтительным для fallback-поиска, из-за чего интернет-поиск оставался привязан к заблокированному сайту.
- Что изменено:
  - добавено распознавание вероятно заблокированных supplier-доменов;
  - антибот-страницы и HTTP-ошибки теперь маркируются явно;
  - при блокировке supplier-домена generic web fallback больше не привязывается к этому домену;
  - для заблокированных supplier-доменов усилены поисковые запросы по `артикул + бренд + название` с отдельными вариантами для `Ozon`, `Яндекс Маркета` и общего web-поиска;
  - в `supplier_parse_comment` сохраняется более понятная причина (`HTTP 401`, `Access blocked`, `fallback_rejected`).
- Что проверить вручную:
  - повторный запуск `Обогатить поставщика` для товаров со ссылками на `sportmaster.ru`;
  - в карточке товара видно понятную причину supplier parse error;
  - при включённых fallback-источниках появляются попытки поиска вне домена поставщика.

### `6f63919` Make product dedupe migration safe during app startup

- Исправлен риск падения приложения на старте в `init_db()`.
- Причина: жёсткая очистка дублей в `products` и создание unique index могли падать на старой или занятой SQLite БД.
- Что изменено:
  - dedupe продуктов по `article` переведён в `best effort`;
  - создание unique index по `article` тоже переведено в безопасный режим;
  - для dedupe используется `id`, а не `rowid`.
- Что проверить вручную:
  - вкладка `Импорт` открывается;
  - каталог не пропадает;
  - импорт новых товаров не падает.

### `0fc51de` Fix Ozon category matching priority after import

- Исправлена деградация автосопоставления категорий Ozon после импорта.
- Причина: матчинг слишком сильно доверял слабым категориям из прайса поставщика и локальной инференции.
- Что изменено:
  - главным сигналом снова стало название товара;
  - слабые значения категорий (`прочее`, `товары`, `каталог` и т.п.) исключены как сильные якоря;
  - память прошлых маппингов больше не применяется по слишком общим ключам;
  - вместо жёсткой блокировки добавлен мягкий штраф за слабое совпадение.
- Что проверить вручную:
  - импорт того же старого файла;
  - количество товаров с Ozon category после импорта;
  - корректность категорий в `Каталог` и `Карточка`.

### `6cf14f0` Fix Excel import auto-header detection regression

- Исправлена регрессия автоопределения листа и строки заголовка в импорте Excel.
- Причина: эвристика заголовка стала выбирать строку, похожую на header, но дающую мало валидных товарных строк.
- Что изменено:
  - усилен scoring строки заголовка;
  - автоимпорт теперь проверяет несколько кандидатов и выбирает тот, где больше реально импортируемых строк;
  - preview в разделе `Импорт` использует более строгую рекомендацию строки.
- Что проверить вручную:
  - повторный импорт старого файла на ~500 артикулов;
  - совпадает ли число импортируемых строк с ожидаемым;
  - не ломается ли ручной режим выбора листа/заголовка.
