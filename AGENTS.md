# AGENTS.md

Этот файл обязателен для любого агента, который продолжает работу над PIM.
Перед изменениями нужно прочитать:

- `README.md`
- `PIM_METHOD.md`
- `PROJECT_CONTEXT.md`
- этот файл `AGENTS.md`

## 1. Главная логика проекта

- `Ozon` — эталон категорий и атрибутов.
- Категории и атрибуты Ozon приоритетнее legacy-категорий из прайсов поставщиков.
- Один товар = одна мастер-карточка.
- Любой новый шаблон клиента расширяет мастер-карточку, а не создаёт отдельный товар.

## 2. Что нельзя ломать

- Импорт Excel поставщика.
- Сохранность каталога между перезапусками.
- Ozon sync и Ozon category matching.
- Клиентские шаблоны, mapping rules и template profiles.
- Source tracking и source priority.
- Конвертацию единиц измерения.
- Мульти-фото: `image_url`, `main_image`, `gallery_images`.

## 3. Обязательные правила разработки

- Перед правками проверь текущий поток: `Импорт -> Каталог -> Карточка -> Клиентский шаблон -> Ozon`.
- Не удаляй существующие данные и не делай разрушительных миграций при старте приложения.
- Любая миграция БД должна быть `best effort`, если есть риск уронить запуск.
- Любой новый источник данных должен быть совместим с `source priority`.
- Любой новый AI-функционал не должен перетирать `manual` без явного решения пользователя.

## 4. Обязательные записи

После каждого заметного изменения агент обязан обновить записи:

1. `PROJECT_CONTEXT.md`
2. `README.md`, если изменился пользовательский сценарий
3. `CHANGELOG.md`

Если был фикс регрессии, это должно быть записано отдельно в `CHANGELOG.md` с датой и краткой причиной.

## 5. Формат записей в CHANGELOG

Для каждой заметной итерации фиксировать:

- дата;
- commit hash, если уже есть;
- что изменено;
- что именно было сломано или почему делалась правка;
- что проверить вручную.

## 6. Что проверять после каждой правки

Минимальный smoke-check:

1. `Импорт` открывается и не падает.
2. Excel импортирует строки в каталог.
3. `Каталог` открывается, фильтры работают.
4. `Карточка` открывается, можно выбрать товар.
5. Если правка касается Ozon:
   - Ozon page открывается;
   - category matching не деградировал.
6. Если правка касается шаблонов:
   - профиль виден;
   - preview и export не падают.

## 7. Приоритет действий

Если есть конфликт между "красиво" и "надёжно":

- сначала надёжно;
- потом удобно;
- потом красиво.

Если есть конфликт между supplier-категорией и Ozon-категорией:

- приоритет у Ozon.

Если есть конфликт между слабым автоисточником и ручным значением:

- приоритет у `manual`.
